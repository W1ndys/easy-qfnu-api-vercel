# 成绩页面自定义绩点计算功能

## 需求描述

在成绩查看页面增加自定义绩点计算功能：每个成绩卡片可勾选，勾选变化时实时显示已勾选课程的加权平均绩点、总学分、课程数，统计区域始终置顶显示在屏幕上。

## 现状分析

### 涉及文件

| 文件 | 角色 | 是否修改 |
|------|------|----------|
| `frontend/src/views/GradeView.vue` | 成绩页面主组件 | **是（唯一修改文件）** |
| `frontend/src/api/zhjw.js` | API 调用 `getGrade()` | 否 |
| `frontend/src/components/AppLayout.vue` | 页面布局壳，sticky header `z-20` | 否（参考其 sticky 样式） |
| `model/grade.go` | 后端成绩数据结构 | 否 |
| `services/zhjw/grade.go` | 后端 GPA 计算逻辑 | 否 |

### 当前实现

- **GradeView.vue**（271 行）：筛选区 → 3 列统计卡片（总加权绩点/总学分/课程总数）→ 按学期分组的成绩列表
- 后端返回 `grades` 数组，每项含 `credit`（学分，string）和 `gpa`（绩点，string）
- 后端 GPA 计算公式：`加权绩点 = Σ(gpa × credit) / Σ(credit)`，保留 2 位小数
- 项目无 checkbox 组件，全部使用 Tailwind 内联样式
- 唯一 sticky 元素是 AppLayout header（`sticky top-0 z-20 bg-white/95 backdrop-blur`）

### 依赖关系

- 纯前端交互功能，**不依赖后端 API 变更**
- 复用现有 `gradeData.value.grades` 数据，无需额外请求
- 沿用项目现有 Tailwind 样式规范

## 实现方案

### Step 1: 添加响应式状态

在 `<script setup>` 中新增：

```js
const selectedKeys = ref(new Set())   // 已选课程的唯一 key 集合
const customCalcMode = ref(false)     // 是否开启自定义计算模式
```

课程唯一 key：`${grade.semester}-${grade.course_code}-${grade.course_name}`（与现有 `v-for :key` 一致）

### Step 2: 添加计算 computed

```js
const customStat = computed(() => {
  if (selectedKeys.value.size === 0) {
    return { weightedGpa: '0.00', totalCredits: '0.00', courseCount: 0 }
  }
  let weightedSum = 0, totalCredits = 0, courseCount = 0
  for (const g of gradeData.value.grades) {
    const key = `${g.semester}-${g.course_code}-${g.course_name}`
    if (!selectedKeys.value.has(key)) continue
    const credit = parseFloat(g.credit) || 0
    const gpa = parseFloat(g.gpa) || 0
    if (credit > 0) {
      weightedSum += gpa * credit
      totalCredits += credit
      courseCount++
    }
  }
  return {
    weightedGpa: totalCredits > 0 ? (weightedSum / totalCredits).toFixed(2) : '0.00',
    totalCredits: totalCredits.toFixed(2),
    courseCount
  }
})
```

### Step 3: 筛选区添加"自定义计算"开关

在筛选区"显示模式"下方新增一个按钮，样式与现有查询按钮一致：

- 开启时文案"关闭自定义计算"，关闭时文案"开启自定义计算"
- 关闭时清空 `selectedKeys`

### Step 4: 添加 sticky 置顶统计栏

在筛选区之后、原有统计卡片之前插入 sticky 统计栏：

- **定位**：`sticky top-[49px] z-10`（AppLayout header 高度约 49px，z-index 低于 header 的 z-20）
- **样式**：`bg-white/95 backdrop-blur rounded-xl shadow-sm`，与 header 毛玻璃效果一致
- **内容**：3 列 grid 展示加权绩点/总学分/已选课程数 + 全选/取消按钮
- **可见性**：仅 `customCalcMode === true` 时显示

### Step 5: 成绩卡片添加 checkbox

在每张成绩卡片中：

- 仅 `customCalcMode` 为 true 时显示 checkbox
- checkbox 使用 `<div>` 模拟：`w-5 h-5 rounded-full border-2 flex-shrink-0`
  - 未选中：`border-gray-300`
  - 选中：`bg-blue-500 border-blue-500` + 白色 SVG 对勾
- 点击整个卡片即可切换选中状态（提升触控体验）
- 选中时卡片边框变为 `border-blue-400`

### Step 6: 全选/取消全选

sticky 栏提供操作按钮：
- **全选**：将当前 `gradeData.value.grades` 中所有课程 key 加入 `selectedKeys`
- **取消全选**：清空 `selectedKeys`

### Step 7: 查询时重置选择

在 `fetchGrades()` 函数中，查询成功后清空 `selectedKeys`，保持 `customCalcMode` 状态不变。

## 交互设计

| 场景 | 行为 |
|------|------|
| 点击"开启自定义计算" | 每张卡片左侧出现 checkbox，置顶栏出现（初始数据为 0） |
| 勾选/取消勾选课程 | 置顶栏数据实时更新 |
| 滚动页面 | 置顶栏始终固定在 header 下方可见 |
| 点击"全选" | 所有课程被选中，置顶栏显示全部数据 |
| 点击"取消全选" | 清空选择，置顶栏归零 |
| 点击"关闭自定义计算" | checkbox 消失，置顶栏消失，清空所有选择 |
| 重新查询成绩 | 清空选择，customCalcMode 保持不变 |
| 无课程被选 | 置顶栏显示 0.00 / 0.00 / 0 门 |

## 风险评估

| 风险点 | 等级 | 说明 |
|--------|------|------|
| 后端兼容性 | 无风险 | 纯前端功能，不修改后端 |
| 计算精度 | 低风险 | 与后端使用相同公式，JavaScript 浮点精度用 toFixed(2) 对齐 |
| 移动端 sticky | 无风险 | `position: sticky` 在所有现代移动浏览器均支持 |
| 性能 | 无风险 | computed 遍历 grades 数组（通常 < 200 条），开销可忽略 |
| 样式冲突 | 无风险 | 沿用项目现有 Tailwind class 规范 |

## 验证方法

1. `task dev:frontend` 启动前端开发服务器
2. 进入成绩查询页面，查询成绩
3. 点击"开启自定义计算"，确认 checkbox 出现、置顶栏出现
4. 勾选若干课程，验证置顶栏数值实时更新
5. 手动验算：`Σ(gpa×credit) / Σ(credit)` 是否与显示一致
6. 滚动页面，确认置顶栏始终固定在 header 下方
7. 点击"全选"，验证所有课程选中且数值正确
8. 点击"取消全选"，验证归零
9. 关闭自定义计算，确认 UI 完全恢复原状
10. 重新查询成绩，确认选择被清空但模式保持

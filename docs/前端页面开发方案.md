# 前端页面开发方案

## 一、项目现状

### 1.1 技术栈

- **框架**: Vue 3 (Composition API + `<script setup>`)
- **构建工具**: Vite 5
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP 客户端**: Axios
- **样式**: Tailwind CSS 3
- **部署**: Vercel (SPA 模式，`/api/*` 转发至 Go Serverless)

### 1.2 已完成部分

- 后端全部 API 已就绪（登录、成绩、课表、考试、选课、培养方案）
- 前端 API 调用函数已封装完毕（`src/api/zhjw.js`）
- Pinia 用户状态管理已实现（cookie 持久化至 localStorage）
- 登录页面已完成（`src/views/LoginView.vue`）
- 路由仅有 `/login` 一个页面

### 1.3 待开发部分

- 主页（功能入口网格布局）
- 成绩查询页
- 课表查询页
- 考试安排查询页
- 选课结果查询页
- 培养方案查询页
- 全局布局组件（导航栏、返回逻辑）
- 路由守卫（登录态校验）
- 全局 401 拦截（cookie 过期自动跳转登录）

## 二、整体设计思路

### 2.1 页面结构

```
/login          → 登录页（已有，需小幅改造：登录成功后跳转主页）
/home           → 主页（功能网格入口）
/grade          → 成绩查询
/schedule       → 课表查询
/exam           → 考试安排
/selection      → 选课结果
/course-plan    → 培养方案
```

### 2.2 布局方案

采用移动优先设计，所有页面以 `max-w-lg mx-auto` 为容器，居中显示，桌面端不会过度拉伸。

页面分为两类：
- **主页**: 无顶部导航栏，独立全屏布局，网格展示功能入口
- **子页面**: 统一顶部栏（含返回按钮 + 页面标题），下方为内容区域

### 2.3 交互流程

```
用户打开应用
  ├─ 未登录 → 路由守卫拦截 → 跳转 /login
  │           └─ 登录成功 → 跳转 /home
  └─ 已登录 → 进入 /home
               └─ 点击功能卡片 → 进入对应子页面
                  └─ 自动请求数据 → 展示结果
                  └─ 若 401 → 清除 cookie → 跳转 /login
```

## 三、开发阶段划分

### 第一阶段：地基 — 全局基础设施

> 目标：搭建所有页面共用的基础设施，后续页面开发可以直接复用。

#### 1.1 创建全局布局组件 `AppLayout.vue`

**位置**: `src/components/AppLayout.vue`

**功能**:
- 顶部栏：左侧返回箭头（点击 `router.back()`）、中间页面标题（通过 prop 传入）
- 顶部栏固定在页面顶部（`sticky top-0`），内容区域可滚动
- 顶部栏高度约 48px，白色背景，底部加细线分隔
- 内容区域通过 `<slot>` 插入

**Props**:
- `title`: String，页面标题
- `showBack`: Boolean，默认 true，是否显示返回按钮

#### 1.2 Axios 响应拦截器增加 401 处理

**修改文件**: `src/api/request.js`

**改动内容**:
- 在响应拦截器中检测 `response.data.code === 401`
- 命中时调用 `useUserStore().clearCookie()`，然后 `router.push('/login')`
- 需要将 router 实例导出或通过其他方式在拦截器中访问

**注意**: Pinia store 在拦截器中使用时需在 app 创建后才能调用。可以延迟导入或采用事件机制。一个简单方案是在拦截器中直接操作 `localStorage` 并使用 `window.location.href = '/login'` 进行硬跳转，避免循环依赖问题。

#### 1.3 路由守卫

**修改文件**: `src/router/index.js`

**改动内容**:
- 添加 `beforeEach` 全局前置守卫
- 逻辑：如果目标路由不是 `/login`，且 localStorage 中无 `zhjw_cookie`，则重定向到 `/login`
- 如果已登录且访问 `/login`，重定向到 `/home`
- 将 `/` 的 redirect 从 `/login` 改为 `/home`

#### 1.4 改造登录页

**修改文件**: `src/views/LoginView.vue`

**改动内容**:
- 登录成功后，将 `alert('登录成功！')` 替换为 `router.push('/home')`
- 在页面 `onMounted` 中检查如果已登录则直接跳转 `/home`（与路由守卫配合）

- 右上角：退出登录按钮（图标或文字均可）
- 主体区域：2 列网格（移动端 2 列，平板及以上可 3 列）
- 底部：版本号或备注信息（可选）

**功能卡片列表**（共 5 个）:

| 卡片名称 | 图标建议 | 路由目标 | 说明 |

### 第三阶段：砌墙 — 各功能页面实现

> 目标：逐个实现每个子页面的数据请求与展示。每个页面遵循相同模式：进入页面 → 自动请求 → 展示数据/错误。

#### 通用页面模式

每个子页面的结构一致：

```
<AppLayout title="页面标题">
  <!-- 筛选条件区域（如有）-->
  <!-- 加载状态 -->
  <!-- 错误提示 -->
  <!-- 数据展示 -->
  <!-- 空状态 -->
</AppLayout>
```

通用状态管理（每个页面内部 ref）：
- `loading`: Boolean，是否加载中
- `error`: String，错误信息
- `data`: 接口返回的数据

#### 3.1 成绩查询页 `GradeView.vue`

**页面行为**:
- 进入页面后自动调用 `getGrade(cookie)` 获取全部成绩
- 支持筛选：学期（term）、课程类型（course_type）、课程名称（course_name）

**筛选区域**:
- 学期选择：下拉框，选项从返回的成绩数据中提取去重的 `semester` 字段动态生成
- 课程类型：下拉框，选项为后端支持的 16 种类型（见下方映射表）
- 课程名称：文本输入框，支持模糊搜索
- 显示模式：切换按钮，"全部成绩" / "最好成绩"
- 点击"查询"按钮重新请求

**课程类型映射**:

| ID | 名称 |
|----|------|
| 01 | 公共课 |
| 02 | 公共基础课 |
| 03 | 专业基础课 |
| 04 | 专业课 |
| 05 | 专业选修课 |
| 06 | 公共选修课 |
| 07 | 专业任选课 |
| 08 | 实践教学环节 |
| 09 | 公共任选课 |
| 10 | 教师教育基础课程（必修） |
| 11 | 专业必修课 |
| 12 | 学科基础必修课 |
| 13 | 专业方向限选课 |
| 14 | 考试报名虚拟课程 |
| 15 | 教师教育选修课程 |
| 16 | 公共必修课 |

**数据展示**:

顶部统计卡片区（横向滚动或网格）：
- 总加权绩点（`total_stat.weighted_gpa`，保留两位小数）
- 总学分（`total_stat.total_credits`）
- 课程总数（`total_stat.course_count`）

成绩列表（卡片式，每条成绩一张卡片）：
- 卡片左侧/顶部：课程名称（加粗）、课程性质标签
- 卡片右侧/底部：分数（大字号，突出显示）、学分、绩点
- 学期信息作为分组标题（可折叠）

**API 调用**:
- `getGrade(cookie)` — 无参数时获取全部
- 带筛选时需修改 `src/api/zhjw.js` 中的 `getGrade` 函数，支持传入查询参数 `{ params: { term, course_type, course_name, display_type } }`

**需要修改 `src/api/zhjw.js`**:
- `getGrade(cookie, params)` 增加第二个参数，将 params 作为 query string 传入

#### 3.2 课表查询页 `ScheduleView.vue`

**页面行为**:
- 进入页面后以当天日期自动调用 `getClassSchedules(cookie)` 获取课表
- 支持切换日期查看不同周的课表

**筛选区域**:
- 日期选择器：`<input type="date">`，默认值为今天
- 切换日期后自动重新请求

**数据展示**:
- 顶部信息栏：显示当前周次（`currentWeekRaw`，如"第18周/20周"）
- 课程列表按节次排序，每门课一张卡片
- 卡片信息：
  - 课程名称（加粗）
  - 上课时间：`星期X 第X-X节`（从 `timeParsed` 格式化）
  - 上课地点（`location`）
  - 学分（`credit`）
  - 上课班级（`classes`）
  - 课程类别标签（`category`）

**空状态**: 当天无课时显示"今日无课"提示

**需要修改 `src/api/zhjw.js`**:
- `getClassSchedules(cookie, params)` 增加第二个参数，支持传入 `{ date }` 查询参数

#### 3.5 培养方案页 `CoursePlanView.vue`

**页面行为**:
- 进入页面后自动调用 `getCoursePlan(cookie)` 获取培养方案
- 数据量较大，需注意渲染性能

**数据展示**:

顶部信息区：
- 培养目标（`objectives`）— 可折叠展示，默认收起

- 每个课程组为一个可折叠的手风琴（Accordion）卡片

- 每门课程显示：
  - 课程名称（`course_name`）
  - 修读状态标签（`status`，如"已修(优)"用绿色，"未修"用灰色）
  - 学分（`credits`）
  - 学时（`hours`）
  - 开课学期（`term`）
  - 课程属性（`course_attr`）

**空状态**: "暂无培养方案数据"

### 第四阶段：封顶 — API 层适配与状态优化

> 目标：完善 API 调用函数，增加全局加载、错误处理等体验细节。

#### 4.1 修改 API 调用函数

**修改文件**: `src/api/zhjw.js`

需要让各查询函数支持可选的查询参数：

| 函数 | 当前签名 | 修改后签名 | 新增 params |
|------|---------|-----------|------------|
| `getGrade` | `(cookie)` | `(cookie, params={})` | `term, course_type, course_name, display_type` |
| `getClassSchedules` | `(cookie)` | `(cookie, params={})` | `date` |
| `getExamSchedules` | `(cookie)` | `(cookie, params={})` | `term` |
| `getSelectionResults` | `(cookie)` | `(cookie, params={})` | `term` |
| `getCoursePlan` | `(cookie)` | 不变 | 无参数 |

修改方式：在 `api.get()` 调用中增加 `params` 字段。

#### 4.2 统一错误提示

各页面在 catch 中统一处理错误：
- 网络错误：显示"网络异常，请检查网络连接"
- 401 错误：由拦截器统一处理跳转（第一阶段已实现）
- 404 错误（无数据）：显示对应的空状态提示
- 其他错误：显示后端返回的 `msg` 字段

#### 4.3 加载状态优化

- 请求过程中按钮禁用，显示 loading 动画
- 使用 Tailwind 的 `animate-pulse` 实现骨架屏效果（可选）
- 下拉刷新支持（可选，移动端体验优化）

### 第五阶段：装修 — 视觉与体验打磨

> 目标：优化移动端体验、完善视觉细节。

#### 5.1 移动端适配要点

- 所有页面容器使用 `max-w-lg mx-auto px-4`，保证内容不会在大屏上过度拉伸
- 卡片使用 `rounded-xl` 圆角 + `shadow-sm` 轻阴影
- 按钮和可点击区域最小高度 44px（符合移动端触摸标准）
- 输入框使用 `text-base`（16px）避免 iOS Safari 自动缩放
- 使用 `safe-area-inset` 处理刘海屏底部安全区域

#### 5.2 主页动效

- 功能卡片点击时有轻微缩放反馈（`active:scale-95 transition-transform`）
- 页面切换使用 Vue Router 的 `<Transition>` 组件添加淡入效果

#### 5.3 色彩规范

基于现有登录页的蓝色主题延续：
- 主色：`blue-600`（按钮、链接、活跃状态）
- 背景色：`gray-50` 或 `gray-100`
- 卡片背景：白色
- 文字主色：`gray-800`
- 文字辅助色：`gray-500`
- 成功/已修：`green-500`
- 警告/未修：`yellow-500`
- 错误：`red-500`

#### 5.4 标签样式

各页面中的标签（课程性质、修读状态等）使用统一的药丸样式：
- 小号圆角标签（`text-xs px-2 py-0.5 rounded-full`）
- 不同类别使用不同底色区分

## 六、后端 API 接口速查

> 供前端开发时快速参考，所有接口基础路径为 `/api`。

### 6.1 登录（无需认证）

```
POST /v1/zhjw/login
Body: { "username": "学号", "password": "密码" }
成功: { code: 200, data: { cookie: "..." } }
失败: { code: 1, msg: "登录失败: ..." }
```

### 6.2 成绩查询

```
GET /v1/zhjw/grade
Header: Authorization: <cookie>
Query: term, course_type, course_name, display_type (均可选)
返回: { code: 200, data: { grades: [...], total_stat, year_stats, semester_stats } }
```

### 6.3 课表查询

```
GET /v1/zhjw/schedule
Header: Authorization: <cookie>
Query: date (可选，格式 2026-01-01)
返回: { code: 200, data: { currentWeekRaw: "...", courses: [...] } }
```

### 6.4 考试安排

```
GET /v1/zhjw/exam
Header: Authorization: <cookie>
Query: term (可选，格式 2024-2025-1)
返回: { code: 200, data: [...] }
```

### 6.5 选课结果

```
GET /v1/zhjw/selection
Header: Authorization: <cookie>
Query: term (可选，格式 2024-2025-1)
返回: { code: 200, data: [...] }
```

### 6.6 培养方案

```
GET /v1/zhjw/course-plan
Header: Authorization: <cookie>
无查询参数
返回: { code: 200, data: { objectives, description, groups: [...] } }
```

### 6.7 通用错误码

| code | 含义 | 前端处理 |
|------|------|---------|
| 200 | 成功 | 正常展示数据 |
| 1 | 系统繁忙/通用错误 | 显示 msg 内容 |
| 401 | cookie 过期 | 清除本地 cookie，跳转登录页 |
| 404 | 无数据 | 显示空状态提示 |
| 502 | 上游系统无响应 | 显示"教务系统暂时无法访问" |
| 1001 | 参数错误 | 显示 msg 内容 |
